# 2026-06-04 币种编辑功能检查记录

## 新增问题

- 编辑币种时，保存接口会更新字段，但调整方式、调整值、官方汇率变更后没有在保存阶段重新计算生效汇率，导致页面上部分配置看起来无效。
- 百分比调整值的后端语义不符合页面直觉，原逻辑中 `1` 会被当成 `100%`，现在调整为 `1 = 1%`。
- 编辑弹窗复用同一个 `ModalForm` 时，切换不同行或新增/编辑状态可能保留旧表单初始值。
- 基准币种业务上固定人民币，但编辑表单仍允许选择其他币种，容易造成误解。
- 币种列表和汇率历史抽屉的汇率展示固定使用 8 位小数，没有按币种的汇率精度展示。

## 已修复问题

- `FinanceCurrencyServiceImpl` 在新增和编辑币种时统一计算生效汇率。
- 调整方式为 `MANUAL` 时，必须填写生效汇率；其他调整方式由后端按官方汇率和调整值计算。
- `PERCENT_UP` / `PERCENT_DOWN` 的调整值按人类百分比处理，例如 `1` 表示 `1%`。
- 基准币种后端固定为 `CNY`，前端编辑表单固定展示人民币。
- 编辑弹窗增加当前币种 key 和打开时重置表单，避免切换行时残留旧值。
- 表单按调整方式控制字段：人工维护才允许填写生效汇率；百分比和固定加减才允许填写调整值。
- 币种列表和汇率历史抽屉按当前币种的汇率精度格式化官方汇率和生效汇率。
