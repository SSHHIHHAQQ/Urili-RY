# 商城商品编辑页与样例数据更新方案

日期：2026-06-05

状态：已按用户确认的设计方向实现并完成前端验证。

## 1. 本次更新目标

在首版“商城商品列表”基础上继续优化管理端录入体验：

- 新增/编辑商品从抽屉改为应用内独立页面；不强制新标签页，用户可自行按住 Ctrl 点击新增入口打开新页面。
- 新增页隐藏系统 SPU，编辑页只读展示系统 SPU。
- 商品标题拆为商品中文标题和商品英文标题，二者均必填。
- 商品分类搜索尽量使用 Ant Design 原生 TreeSelect 行为，只按当前节点标题匹配。
- 图片上传框改为图片槽位样式，固定 8 个 SPU 图片槽；第 1 张为必填主图，第 2 张仅标识为尺寸图提示，其余槽位空白展示。
- SKU 区域改为“规格属性 -> 规格值 -> 生成 SKU 矩阵 -> 表格补充价格/图片/状态”的录入方式。
- 增加详情图文能力：SPU 通过模块化详情内容保存文本段落、图片模块、图文模块和参数表模块。
- 新增/编辑 SKU 表隐藏系统 SKU，详情页和列表查看场景保留系统 SKU。
- 新增“发货仓库（预留）”UI，同一 SPU 可选多个同国家仓库；币种由所选仓库国家推导，未选择仓库时 SKU 币种列显示 `-`。
- 向运行库写入一组真实感样例商品，便于检查页面效果。

## 2. 数据库变更

### 2.1 `product_spu.product_name`

保留现有字段，语义明确为“商品中文标题”。

### 2.2 新增 `product_spu.product_name_en`

- 类型：`varchar(255)`
- 默认值：空字符串
- 业务含义：商品英文标题。
- 必填规则：后端保存时必填；存量数据迁移时先用中文标题回填，避免旧数据为空。

### 2.3 新增 `product_spu.detail_content`

- 类型：`text`
- 业务含义：商品详情内容，保存 JSON 结构，当前支持文本段落、图片模块、图文模块和参数表模块。
- 是否必填：否。

### 2.4 详情图片

本轮不新增详情图片表，也不再单独把详情图片拆成 `product_image` 的 `DETAIL` 记录；详情模块里的图片 URL 随 `detail_content` JSON 一起保存。后续如果需要详情图独立排序、独立审核或独立素材库，再重新设计存储边界。

## 3. 样例数据策略

使用当前远端 `fenxiao` 运行库已有基础数据：

- 卖家：`seller_id=5`，卖家编号 `S20260603202422748237`，卖家名称 `跨新科技`。
- 分类：
  - `600` 棒球帽
  - `565` 跑步鞋
  - `588` 通勤双肩包
  - `446` T恤
- 币种：`CNY`、`USD`。

样例数据只插入 `system_spu_code` 以 `SPUDEMO20260605` 开头、`system_sku_code` 以 `SKUDEMO20260605` 开头的演示商品，便于后续识别和清理。

## 4. 不做的事

- 不新增库存表。
- 不新增审核表。
- 不改变 seller/buyer 端权限。
- 不把系统 SPU 暴露为新增页输入项。
- 不让商品分类搜索继续匹配整棵子树文本。
- 不保存发货仓库绑定；仓库真实模块落地后再替换临时选项并补持久化字段。
