# 类目属性模板筛选增强执行记录

日期：2026-06-04

## 1. 目标

在商品属性配置的“类目属性模板”页签中补齐两层筛选能力：

- 左侧类目树支持快速查找类目，例如搜索“牛仔裤”后快速定位该类目。
- 右侧类目规则表支持按属性规则字段筛选，便于检查本类目规则和继承预览。
- 类目树支持“展开全部 / 收起全部”。

## 2. 实现范围

本次只修改管理端前端 `react-ui`，未修改后端接口、数据库表、菜单权限或 SQL。

## 3. 修改文件

- `react-ui/src/pages/Product/Attribute/components/CategoryAttributeTemplate.tsx`
- `react-ui/src/pages/Product/Attribute/components/CategoryTreeFilterPanel.tsx`
- `react-ui/src/pages/Product/Attribute/components/categoryAttributeFilterUtils.ts`
- `react-ui/src/pages/Product/Attribute/components/categoryAttributeColumns.tsx`

## 4. 功能说明

### 4.1 类目树筛选

左侧类目树新增：

- 类目关键词：匹配类目名称、类目编码和完整路径。
- 类目状态：正常类目、停用类目、全部状态。
- 类目层级：按当前数据中的层级动态生成。
- 只看末级类目。
- 展开全部、收起全部。

筛选后保留命中类目的祖先节点，避免搜索结果变成孤立节点。当前选中类目如果被筛选条件隐藏，会清空右侧规则数据，避免继续展示旧类目的规则。

### 4.2 类目规则筛选

本类目规则表新增本地筛选：

- 关键词：匹配属性名称、属性编码。
- 属性类型。
- 属性分组。
- 是否必填。
- 是否可筛选。
- 状态。

继承预览表新增本地筛选：

- 关键词：匹配属性名称、属性编码。
- 来源范围：本类目、继承上级。
- 来源类目。
- 属性类型。
- 属性分组。
- 是否必填。
- 是否可筛选。
- 状态。

## 5. 工程处理

- 复用现有 `getPersistedProTableSearch(...)`，保持三端管理表格筛选区的统一布局和折叠记忆。
- 复用 `SEARCHABLE_SELECT_PROPS`，所有下拉筛选支持模糊搜索。
- 将类目筛选面板、筛选工具函数、表格列配置拆出，避免继续扩大主页面组件。
