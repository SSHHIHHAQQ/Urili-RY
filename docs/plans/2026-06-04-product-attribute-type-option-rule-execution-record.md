# 商品属性类型与选项来源规则修正执行记录

## 背景

商品属性库原实现允许属性类型与选项来源自由组合，导致文本、数字、布尔、日期等非选择型属性也能配置属性自定义选项或若依字典；同时属性类型包含当前不需要的文件类型，字典类型需要手工输入编码，操作人员容易填错。

## 本次规则

| 属性类型 | 选项来源规则 | 字典类型规则 |
| --- | --- | --- |
| `TEXT` 文本 | 固定 `NONE`，页面不展示选项来源 | 留空 |
| `NUMBER` 数字 | 固定 `NONE`，页面不展示选项来源 | 留空 |
| `BOOLEAN` 布尔 | 固定 `NONE`，页面不展示选项来源 | 留空 |
| `DATE` 日期 | 固定 `NONE`，页面不展示选项来源 | 留空 |
| `SINGLE_SELECT` 单选 | 只能选 `ATTRIBUTE_OPTION` 或 `SYS_DICT` | 选 `SYS_DICT` 时必选若依字典类型 |
| `MULTI_SELECT` 多选 | 只能选 `ATTRIBUTE_OPTION` 或 `SYS_DICT` | 选 `SYS_DICT` 时必选若依字典类型 |

`FILE` 文件属性类型本阶段删除，不再出现在属性库表单和导入模板说明中。

## 已完成改动

- 前端属性类型选项删除 `FILE`。
- 前端新增 `isOptionAttributeType(...)`，统一判断单选/多选是否需要选项来源。
- 新增/编辑商品属性弹窗：
  - 文本、数字、布尔、日期不展示选项来源和字典类型。
  - 单选、多选才展示选项来源。
  - 单选、多选默认使用属性自定义选项。
  - 选择若依字典后才展示字典类型。
  - 字典类型改为若依字典类型下拉，展示 `字典名称（dict_type）`，保存 `dict_type`。
- 属性库操作列恢复为 `编辑 / 选项 / 删除` 直接展示，不再将单独删除放进“更多”。
- 后端商品属性新增/编辑增加同样规则校验，避免绕过前端提交非法组合。
- 后端属性选项新增/编辑增加保护：只有 `ATTRIBUTE_OPTION` 属性可以维护自定义选项。
- 商品属性导入校验同步限制类型与选项来源组合。
- 商品属性导入模板字段说明增加同样规则，并补充若依字典示例。
- 复用台账已登记属性类型判断复用规则。

- 本次没有执行 DDL。
- 本次没有执行成功的业务数据新增或更新；非法组合接口验证被后端拒绝。

## 文件规模判断

- `react-ui/src/pages/Product/Attribute/components/AttributeLibrary.tsx` 已超过 400 行，当前仍保持不拆分。
