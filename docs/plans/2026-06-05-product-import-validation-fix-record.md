# 商品配置导入校验问题修复记录

记录时间：2026-06-05 16:13:40 +08:00

## 目标

排查商品属性导入弹窗中“校验看起来没效果、确认导入不可用”的问题，确认商品分类、商品属性、属性选项三类导入的前端处理是否能正确承接后端校验结果。

## 排查结论

- 后端导入服务已有预校验链路：`importPreview` 返回行级校验结果，`importData` 在真正写入前也会先跑一遍校验，校验不通过不会执行导入。
- 商品属性导入的行级错误由 `ProductImportResult.messages` 承载，包含行号、动作、状态和说明。
- 前端问题在导入弹窗对异常返回和若依 `AjaxResult.warn(...)` 的结果承接不够完整，容易被全局错误处理打断或没有把行级错误展示在弹窗内。
- “确认导入”应只在已选择文件、已经完成校验、总行数大于 0、错误数为 0 时可点击。

## 改动内容

- `react-ui/src/pages/Product/components/ProductImportModal.tsx`
  - 校验开始时清空旧结果，避免旧文件结果误导。
  - 对预览和确认导入都增加异常兜底，尽量从异常对象中提取后端行级校验结果并展示。
  - 将确认导入条件收紧为：有文件、有预览结果、总行数大于 0、错误数为 0。
  - 空模板或错误文件校验后会显示错误明细，确认导入保持禁用。
  - 有效文件校验通过后会显示新增/更新明细，确认导入恢复可用。
  - 补充前端文件类型校验，只允许 `.xls` / `.xlsx`。
  - 清理 Ant Design v6 弃用属性，减少调试时控制台干扰。
- `react-ui/src/services/product/product.ts`
  - 商品分类、商品属性、属性选项的预览和确认导入接口统一设置 `skipErrorHandler: true`，由导入弹窗自己处理校验反馈。
