# 商城商品真实感演示数据生成执行记录

## 目标

为管理端「商品管理 / 商城商品列表」生成 20 个真实感 SPU，每个 SPU 带随机数量 SKU，用于观察列表、状态 Tabs、SPU/SKU 展开行、图片、价格区间、规格摘要和尺寸重量展示效果。

## 数据源与影响范围

- 目标环境：当前后端激活的 `druid` 主数据源，对应远端 MySQL 的 `fenxiao` 业务库。
- 配置来源：`RuoYi-Vue/ruoyi-admin/src/main/resources/application.yml` 与 `application-druid.yml`。
- 执行方式：调用后端管理端接口，不直接执行 SQL。
- 登录身份：管理端 `admin`。
- 影响类型：业务数据新增，会写入远端运行库。
- 预计新增：
  - `product_spu`：20 条
  - `product_sku`：62 条
  - `product_image`：约 82 条，包含 SPU 主图与 SKU 主图
  - `product_attribute_value`：按所选类目模板自动填充必填属性，数量取决于当前类目模板
- 不涉及：
  - 不新增或修改类目、属性、字典、卖家、仓库、库存、订单、财务数据
  - 不执行 DDL
  - 不直接修改库存数，库存仍保持后续库存模块同步边界

## 生成规则

- 所有 SPU 绑定已有启用卖家，优先使用当前已有的跨新科技卖家主体。
- 所有 SPU 选择已有启用且可发布的末级类目。
- 币种使用已启用的 `CNY`。
- 系统 SPU / 系统 SKU 由后端接口自动生成。
- 客户 SPU / 客户 SKU 使用 `DEMO-20260605-*` 前缀，避免与既有样例重复。
- 状态覆盖当前 Tabs：待上架、已上架、已下架、停用、草稿、全部。
- 已上架商品按业务流转创建：先新增待上架，再将 SKU 和 SPU 切换为已上架。
- 每个 SPU 的 SKU 数量已随机确定，范围 1-5 个。
