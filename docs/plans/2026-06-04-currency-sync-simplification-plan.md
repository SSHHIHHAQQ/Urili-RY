# 币种汇率同步设置简化方案

日期：2026-06-04

状态：已实施完成。

## 背景

当前币种同步设置按“通用 API 配置”实现，页面暴露了服务商代码、服务商名称、API 地址、认证方式、超时、重试、同步方式、Cron 等技术字段。这个方向不符合当前产品预期。

新的方向应参考领星 WMS 主仓接入：系统为固定服务商提供固定适配逻辑，运营只维护少量业务参数和密钥，不需要理解 API 地址、认证方式、签名、超时、重试等技术细节。

## 目标

将“汇率同步设置”改成固定官方汇率服务接入：

- 固定服务商和固定后端适配器，不在页面让用户配置 API URL。
- 页面只开放必要业务字段。
- 外部返回的汇率仍然定义为官方汇率 `official_rate`。
- 业务实际使用汇率仍然是生效汇率 `effective_rate`。
- 同步时按“汇率基准时间”选择官方汇率时间点。

## 用户已明确规则

假设汇率基准时间设置为 `09:30`：

- 如果外部返回 `09:29`、`09:31`、`09:32`，必须选择 `09:31`。
- 不能选择 `09:29`。
- 核心规则是：取基准时间之后的第一条汇率。

## 推荐字段

| 中文名 | 字段名 | 类型 | 说明 |
| --- | --- | --- | --- |
| 汇率基准时间 | `rate_anchor_time` | `time` | 用于选择官方汇率的目标时间，例如 `09:30:00` |

命名说明：

- 不建议叫 `base_time`，容易和 `base_currency_code` 混淆。
- `rate_anchor_time` 表示“汇率锚定时间”，业务含义比 `sync_time` 更清楚。
- 页面展示文案使用“汇率基准时间”。

## 页面调整

### 删除页面开放字段

以下字段不再让用户配置：

- 服务商代码
- 服务商名称
- API 地址
- 认证方式
- 请求超时
- 重试次数
- 同步方式
- Cron 表达式
- 基准币种

### 保留或新增页面字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 应用名称 | 只读文本 | 固定 `fenxiao` |
| 应用 ID | 只读文本 | 固定应用 ID，非密钥 |
| 接入密钥 | 密码输入 | ShowAPI `appKey`，保存后加密，只展示脱敏值 |
| 汇率基准时间 | 时间选择 | 默认 `09:30:00` |

同步日志表格继续保留，用于查看每次请求和错误信息。

## 后端调整

### 同步配置表

不建议立刻删除旧字段，避免破坏已执行 SQL 和历史配置。第一步采用兼容改造：

- 新增字段：`rate_anchor_time time not null default '09:30:00' comment '汇率基准时间'`
- 旧字段如 `provider_code`、`provider_name`、`api_base_url`、`auth_type`、`request_timeout_ms`、`retry_count`、`schedule_type`、`cron_expression` 暂时保留。
- 后端保存时固定写入服务商信息和技术默认值，不再依赖前端传入。

### 固定适配器

当前 `CurrencyRateSyncClient` 偏通用 API，应改成固定官方汇率适配器，例如：

- `OfficialCurrencyRateClient`
- 或等待真实服务商确定后命名为具体服务商，例如 `XxxCurrencyRateClient`

适配器职责：

- 根据固定 ShowAPI 接口和 `appKey` 请求官方汇率。
- 解析 `showapi_res_body.list`。
- 以 `hui_out` 现汇卖出价作为官方汇率来源。
- 输出可被 finance 模块消费的汇率候选列表。
- 记录脱敏请求日志。

Finance Service 职责：

- 不关心签名和接口细节。
- 只拿到适配器返回的官方汇率候选列表。
- 按 `rate_anchor_time` 选择官方汇率。
- 计算 `effective_rate`。
- 写入 `finance_currency` 和追加 `finance_currency_rate_history`。

## 汇率选择算法

输入：

- `rate_anchor_time = 09:30:00`
- 外部返回某币种的一组官方汇率记录，每条包含：
  - 币种 code
  - 官方汇率
  - 官方汇率时间

处理：

1. 只保留目标币种对应的官方汇率记录。
2. 按官方汇率时间升序排序。
3. 丢弃早于基准时间的记录。
4. 选择基准时间之后的第一条记录。
5. 对 ShowAPI 返回的 `hui_out` 做单位转换：
   - ShowAPI 外汇牌价是“100 外币兑人民币”。
   - 系统保存 `official_rate` 为“1 单位外币兑人民币”。
   - 因此 `official_rate = hui_out / 100`。
   - 人民币 `CNY` 固定为 `1`。
6. 用该记录写入：
   - `official_rate`
   - `official_rate_time`
7. 再按调整规则计算：
   - `effective_rate`
   - `effective_rate_time`
8. 追加汇率历史。

示例：

| 返回时间 | 是否可选 |
| --- | --- |
| 09:29 | 不可选 |
| 09:31 | 可选，且应被选中 |
| 09:32 | 可选，但不是第一条，不选 |

## 已确认问题

1. 如果外部刚好返回 `09:30:00`，允许取这一条，规则按 `>= 09:30:00`。
2. 如果当天没有 `09:30` 之后的汇率，本次同步失败并不更新；本业务每天同步，不跨到下一天取值。
3. 时间比较统一按北京时间 `Asia/Shanghai`。
4. 当前 ShowAPI 接入使用一个 `appKey` 字段；该字段按敏感凭证处理，不写入 Markdown、SQL、日志或前端响应明文。
5. 基准币种固定为人民币 `CNY`，不在页面开放配置。
6. 官方汇率服务商确定为 ShowAPI 银行汇率查询：`https://www.showapi.com/apiGateway/view/105`。

## ShowAPI 接入点

- 汇率查询：`https://route.showapi.com/105-30?appKey={your_appKey}`
- 请求方式：`POST/GET`
- 可选参数：`code`，例如 `USD`；不传则返回全部。
- 返回字段：
  - `day`：发布日期。
  - `time`：发布时间。
  - `code`：币种简码。
  - `hui_out`：现汇卖出价，作为系统官方汇率来源。
  - `zhesuan`：中行折算价，不作为系统官方汇率来源。
- 费用与权限错误以 `showapi_res_code` 和 `showapi_res_error` 返回。
- 页面说明该接口为中国银行外汇牌价延迟数据，更新频率为每 10 秒。

4. 后端把同步逻辑从通用 API 配置改为固定官方汇率适配器。

6. 前端简化同步设置表单，只保留密钥、汇率基准时间、启用状态、备注和只读同步信息。
7. 更新复用台账、目标追踪和数据库执行记录。
8. 运行后端编译、前端类型检查和构建，浏览器验证页面。
