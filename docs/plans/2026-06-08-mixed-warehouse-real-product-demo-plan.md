## 执行方式

优先使用后端管理端商品新增 API 落库，不直接裸写 `product_spu` / `product_sku`：

- 官方仓商品：
  - `warehouseKind=official`
  - SKU 使用来源商品库维度组字段。
  - 后端服务负责派生发货仓库、来源绑定、尺寸重量来源和唯一绑定校验。
- 三方仓商品：
  - `warehouseKind=third_party`
  - `warehouseIds=[4]`

## 图片设计规则

每个 SPU 填满 8 个图片槽位：

1. 主图：清晰展示商品主体。
2. 尺寸图：展示尺寸/规格感，不强制做真实测量标注。
3. 场景图：展示使用场景。
4. 细节图：材质、纹理、五金、车线、纤维等。
5. SKU 色系图：多规格对比。
6. 包装/陈列图：B2B 备货感。
7. 卖点图：用画面表达功能，不在图内写大量文字。
8. 备用轮播图：补足商品列表和预览观感。

详情图文每个商品都使用四类模块：

- 文本段落
- 图片模块
- 图文模块
- 参数表模块

图片生成方式：

- 使用内置 `image_gen`，不走外部 CLI。
- 每个商品先写清楚图片意图，再生成资产。
- 生成后保存到工作区静态目录。
- 对明显跑题、含水印、商品主体不清楚或风格不统一的图片重新生成。

5. 删除这些 SPU 对应的 `product_attribute_value`。
