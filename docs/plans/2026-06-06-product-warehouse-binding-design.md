# 商城商品发货仓库绑定设计
## 目标

把商城商品新增/编辑页的“发货仓库”从前端预留选择改成真实保存。仓库主数据复用现有 `warehouse` 表；商品只保存“当前可发货仓库绑定关系”，库存数量仍由后续库存模块同步和调整，不在商品模块写库存。

## 表设计

### `product_spu_warehouse`

业务目的：记录一个 SPU 允许绑定的发货仓库。一个 SPU 可以绑定多个仓库，但所有绑定仓库的 `settlement_currency` 必须一致，SKU 币种由该币种推导。

业务逻辑：保存商品时删除旧绑定并写入最新绑定；不是库存流水，不做只追加。库存明细、仓库可售数、库存变动由库存模块负责。

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | bigint | 是 | 主键，自增 |
| `spu_id` | bigint | 是 | 商品 SPU ID，关联 `product_spu.spu_id` |
| `warehouse_id` | bigint | 是 | 仓库 ID，关联 `warehouse.warehouse_id` |
| `warehouse_code` | varchar(64) | 是 | 仓库编码快照，便于展示和排查 |
| `warehouse_name` | varchar(200) | 是 | 仓库名称快照，便于展示和排查 |
| `warehouse_kind` | varchar(32) | 是 | 仓库类型快照：`official` / `third_party` |
| `settlement_currency` | varchar(16) | 是 | 仓库结算币种快照，也是商品 SKU 币种来源 |
| `seller_id` | bigint | 是 | 商品卖家 ID 快照，用于校验第三方仓归属 |
| `create_by` | varchar(64) | 否 | 创建人 |
| `create_time` | datetime | 否 | 创建时间 |

索引与约束：

| 约束/索引 | 字段 | 说明 |
| --- | --- | --- |
| 主键 | `id` | 自增主键 |
| 唯一键 | `spu_id, warehouse_id` | 同一 SPU 不重复绑定同一仓库 |
| 普通索引 | `warehouse_id` | 方便后续反查仓库关联商品 |
| 普通索引 | `seller_id` | 方便按卖家排查商品仓库绑定 |
| 普通索引 | `settlement_currency` | 方便校验和查询币种一致性 |

## 保存校验

1. 新增/编辑商品必须选择至少一个发货仓库，否则无法确定 SKU 币种。
2. 所选仓库必须存在且状态为正常。
3. 所选仓库的 `settlement_currency` 必须全部一致。
4. 所选仓库的 `warehouse_kind` 必须全部一致，官方仓不能和三方仓混选。
5. 绑定第三方仓库时，第三方仓库归属卖家必须等于商品绑定卖家；官方仓库不受卖家归属限制。
6. 保存时后端以仓库币种覆盖/校验 SKU 的 `currency_code`，不允许前端自由传币种。

## 前端接线

1. 商品新增/编辑页使用现有仓库接口读取官方仓和第三方仓。
2. 删除“预留”文案，字段名改为“发货仓库”。
3. 新增/编辑页先选择仓库类型：官方仓 / 三方仓，再选择该类型下的仓库。
4. 下拉项展示仓库名称、仓库编码、类型、币种。
5. 选择多个仓库时前端先做同币种、同仓库类型拦截，后端再做最终校验。
6. 编辑商品时回显已绑定仓库；SKU 表格币种显示所选仓库币种。

## 后端接线

1. `ProductSpu` 增加发货仓库绑定列表字段。
2. 商品详情接口返回绑定仓库。
3. 新增/编辑商品保存绑定关系。
4. 商品列表增加“仓库类型”展示，从 `product_spu_warehouse.warehouse_kind` 聚合得到；该字段只表示发货仓绑定类型。
5. 商品列表仍先展示库存模块读取边界：总可售库存、仓库数当前没有库存模块数据时继续显示 `--`。
