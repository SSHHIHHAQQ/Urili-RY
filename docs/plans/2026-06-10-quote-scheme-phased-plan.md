# 报价方案分阶段实施计划

日期：2026-06-10

## 当前结论

报价方案第一版不做“自动最优”算法，也不做完整成本引擎。

当前更稳的方向是先把报价方案做成链路配置中心：先能维护一个方案，能绑定适用买家范围，能选择客户渠道，能引用操作费设置。后续再逐步扩展到 API 试算、成本来源、自动最低价和手工费率。

报价方案本身只整理“哪些对象、哪些渠道、哪些费用配置可以参与后续计算”，不直接承担最终订单的最优决策。

## 拆分原则

- 每个阶段只增加一层能力，避免一次改动表结构、接口、页面、算法和下单链路。
- 每阶段完成后都要能单独验证；如果发现业务口径不对，立刻停止扩展，先修正设计。
- 不为了未来自动化提前做复杂规则引擎；只保留必要字段和接口边界。
- 报价方案引用现有客户渠道，不重新定义渠道体系。
- 自动最优可以后续视为一种特殊客户渠道或下单策略，但不进入第一版。

## 阶段一：基础配对

目标：把“报价方案基础信息 + 适用对象 + 客户渠道 + 操作费设置引用”做出来。

### 页面能力

- 报价方案列表。
- 新增、编辑报价方案基础信息。
- 启用、停用报价方案。
- 进入方案详情后维护配置明细。

### 基础信息

- 报价方案名称。
- 报价方案编码。
- 状态：启用 / 停用。
- 生效开始时间、失效时间。
- 币种。
- 适用对象类型：
  - 全部买家。
  - 买家等级。
  - 指定买家。
- 买家等级使用现有 `buyer_level` 字典。
- 备注。

### 配置明细

第一版只维护最小配对：

- 客户渠道。
- 操作费设置。
- 排序。
- 状态。
- 备注。

如果操作费设置当前还没有真实表和页面，阶段一需要先补一个最小“操作费设置”可选项来源，只保存名称、编码、币种、状态和备注；不在本阶段实现公式计算。

### 不做内容

- 不计算买家最优渠道。
- 不计算平台最低成本。
- 不接物流商 API 试算。
- 不接主仓 / 报价仓 API 试算。
- 不做利润底线。
- 不做运费公式、附加费公式和手工费率明细。
- 不影响订单下单链路。

### 阶段一验收

- 管理端可以打开报价方案菜单。
- 可以新增一个方案。
- 可以绑定买家等级，例如 `L1`。
- 可以给方案添加至少一个客户渠道。
- 可以给客户渠道绑定一个操作费设置。
- 禁止同一方案下重复绑定同一客户渠道。
- 后端、前端权限点和菜单合同明确。
- 生成实施记录 Markdown。

## 阶段二：试算来源接入

目标：在基础配对稳定后，为每条明细补充试算来源。

新增能力：

- 明细上选择试算来源：
  - 履约仓 API。
  - 报价仓 API。
  - 物流商 API。
  - 手工费率，先保留但可不启用。
- 支持维护仓库范围。
- 支持维护系统渠道范围。
- 返回试算结果快照，但不做最终自动决策。

本阶段只回答“这条链路去哪里拿价格”，不回答“哪条最优”。

## 阶段三：买家侧自动最优

目标：把自动最优从报价方案里抽出去，作为下单侧策略使用报价方案提供的候选链路。

新增能力：

- 支持特殊客户渠道或下单策略“自动最优”。
- 下单侧根据报价方案筛出候选客户渠道、仓库和费用来源。
- 计算买家下单应付费用：
  - 运费。
  - 操作费。
  - 出库时产生的附加费。
  - 包材、签名、偏远、超尺寸等购买时费用。
- 不计入产品价格、仓储费、长期库存费或资金类费用。
- 自动选择买家应付总费用最低的结果。
- 订单保存买家选择值和系统解析出的实际客户渠道、仓库、费用快照。

## 阶段四：平台侧成本最优

目标：在买家报价结果已稳定后，再做平台成本决策。

新增能力：

- 同一个客户渠道可对应多个系统渠道、仓库、物流商链路。
- 成本侧支持多来源试算：
  - 主仓负责面单。
  - 平台购买面单后推给主仓。
  - 多物流商同类渠道比价。
  - 报价仓 / 自营仓试算。
  - 后续手工费率。
- 自动选择平台成本最低链路。
- 订单保存成本链路快照。

本阶段不做利润底线。若买家价格低于平台成本，视为运营配置结果，不在系统里强制拦截。

## 停止条件

任一阶段出现以下情况，先停止扩展：

- 发现当前配置无法解释已有客户渠道、系统渠道或物流商渠道关系。
- 需要新增业务表但字段含义、唯一约束、权限点没有确认。
- 阶段一页面需要提前承担阶段三或阶段四的算法逻辑。
- 订单链路需要依赖尚未验证的价格结果。
- 操作费、运费、附加费边界再次变化。

## 下一步建议

下一次实施只做阶段一。

实施前需要先确认两点：

1. 操作费设置是否允许先做最小可选项来源，而不是完整计费公式。
2. 第一版适用对象是否只做“全部买家 / 买家等级 / 指定买家”，暂不做卖家、店铺或平台渠道维度。
