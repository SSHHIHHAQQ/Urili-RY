# 官方仓来源 SKU 行级配对修复记录

## 背景

官方仓商品从三方仓切换后会保留原 SKU。原实现只能通过顶部「选择来源 SKU」批量选择，再按未配对行顺序自动套用来源 SKU，SKU 数量较多时容易错配。

同时，未配对行在官方仓模式下已经禁用长度、宽度、高度和重量输入，但仍显示原有第三方仓 SKU 的尺寸重量，给用户造成这些值已经来自官方来源 SKU 的误解。

## 修复内容

1. SKU 表格的「来源SKU」列新增行内「配对」按钮。
2. 已配对行显示来源 SKU，并提供「更换」「解除」操作。
3. 行内配对弹窗使用单选模式，确认后只作用于当前 SKU 行。
4. 顶部批量选择不再按顺序套到未配对行，只用于批量新增来源 SKU 行或更新已有来源 SKU 行。
5. 官方仓未配对行的长度、宽度、高度和重量显示为 `-`，不再显示旧值；真正配对来源 SKU 后才展示来源尺寸重量并锁定。

## 验证

- `npm run tsc` 通过。
- 浏览器打开官方仓商品编辑页，未配对 SKU 行显示「未配对 + 配对」。
- 未配对行尺寸重量列显示 `-`。
- 点击行内「配对」后弹窗标题为「配对来源 SKU」，选择控件为单选。
- 验证过程未点击保存，未修改远端业务数据。
