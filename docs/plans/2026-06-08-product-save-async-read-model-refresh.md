# 商城商品保存读模型异步刷新优化记录

## 背景

管理端新增/编辑商城商品时，保存接口原先在同一个事务调用链中同步刷新库存概览和来源商品读模型。官方仓商品保存来源 SKU 绑定时，还会同步触发来源商品、来源库存读模型重建，导致用户点击保存后长时间等待。

## 调整内容

1. 商品核心保存与读模型刷新解耦。
   - `insertProduct` / `updateProduct` 只同步完成 SPU、仓库、SKU、属性、图片和来源绑定事实写入。
   - 库存概览刷新改为事务提交后异步执行。
   - 来源配对投影触发的来源商品/来源库存读模型刷新改为事务提交后异步执行。

2. 来源读模型刷新增加变化判断。
   - 仅当来源 SKU、来源维度组、系统 SKU、绑定状态或锁定状态变化时，才同步更新配对投影并登记后置读模型刷新。
   - 绑定内容未变化时，不再因为普通保存重复刷新来源读模型。

3. 增加保存耗时日志。
   - 后端记录 SPU 主表、仓库、SKU/来源绑定、属性、图片、异步刷新登记的分段耗时。
   - 后台读模型刷新完成或失败时记录任务名和耗时。

4. 前端保存体验调整。
   - 编辑保存成功后停留当前页，只提示已保存。
   - 新增保存成功后，如果接口返回新 SPU ID，则切换到编辑态 URL。
   - 不再保存成功后强制跳回商品列表。

## 验证

- `react-ui`: `npm run tsc`
- `RuoYi-Vue`: `mvn -pl product,inventory,ruoyi-admin -am -DskipTests compile`
- `RuoYi-Vue`: `mvn -pl inventory -Dtest=InventoryOverviewRefreshContractTest test`

## 注意

保存接口返回后，库存概览和来源展示数据可能存在短暂延迟。列表页和详情页读取这些读模型时，应接受后台刷新完成前的旧值或空值。
