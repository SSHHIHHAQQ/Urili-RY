# 商品配置大批量测试数据准备方案

日期：2026-06-04

## 1. 目的

为管理端商品分类配置、商品属性配置准备大批量基础测试数据，用于验证数据量超过 200 条时，列表、树、分页、弹窗、导入预览和后续页面交互是否存在性能或展示问题。

## 2. 当前边界

- 本方案当前只准备导入 Excel 文件，并通过导入预览接口做校验。
- 未经再次确认前，不执行确认导入，不写入当前激活数据库。
- 本次不新增表、不改表、不调整菜单权限。
- 本次不做 SKU、多 SKU、库存、价格、商品发布、商品审核和类目属性绑定导入。

## 3. 数据规模

### 商品分类

计划生成 260 条：

- 顶级分类：10 条。
- 二级分组：50 条，每个顶级分类下 5 条。
- 三级叶子类目：200 条，每个二级分组下 4 条。

规则：

- 顶级分类和二级分组因存在子级，系统自动判定为不可发布。
- 三级叶子类目因不存在子级，系统自动判定为可发布。
- 父级分类编码总是位于子级行之前，满足导入规则。
- 状态统一为 `正常`。

### 商品属性

计划生成 220 条：

- 文本属性：60 条。
- 数值属性：60 条。
- 布尔属性：40 条。
- 单选属性：40 条。
- 多选属性：20 条。

规则：

- 状态统一为 `正常`。
- 数值属性填写单位和数值精度。
- 单选、多选属性使用 `ATTRIBUTE_OPTION` 作为选项来源。

### 商品属性选项

计划为 60 条选择型属性生成 300 条选项：

- 单选属性 40 条，每条 5 个选项，共 200 条。
- 多选属性 20 条，每条 5 个选项，共 100 条。

## 4. 编码和回滚口径

所有压测数据统一使用 `loadtest_20260604_` 前缀：

- 分类编码：`loadtest_20260604_cat_...`
- 属性编码：`loadtest_20260604_attr_...`
- 选项编码：`opt_...`

如果后续确认导入后需要回滚，应按以下顺序删除：

1. 删除 `product_attribute_option` 中 attribute code 前缀为 `loadtest_20260604_` 的选项。
2. 删除 `product_attribute` 中 attribute code 前缀为 `loadtest_20260604_` 的属性。
3. 删除 `product_category` 中 category code 前缀为 `loadtest_20260604_` 的分类，先删叶子，再删二级，再删顶级。

回滚前仍需确认当前激活数据库和备份状态。

## 6. 验证方式

生成文件后，先执行以下非写库验证：

- 读取 Excel 行数和 sheet 结构。
- 调用导入预览接口：
  - `/product/admin/categories/importPreview`
  - `/product/admin/attributes/importPreview`
  - `/product/admin/attributes/options/importPreview`
- 预览结果必须无错误后，才能进入确认导入。

当前非写库验证结果：

| 验证项 | 结果 |
| --- | --- |
| Excel 工作簿结构检查 | 通过，表头和前几行内容正确 |
| 公式错误扫描 | 通过，无 `#REF!`、`#DIV/0!`、`#VALUE!`、`#NAME?`、`#N/A` |
| 商品分类导入预览 | 通过，`code = 200`，总计 260 行，新增 260 行，错误 0 行 |
| 商品属性导入预览 | 通过，`code = 200`，总计 220 行，新增 220 行，错误 0 行 |
| 商品属性选项导入预览 | 当前未通过，`code = 601`，总计 300 行，错误 300 行；原因是依赖的 60 条选择型属性尚未写入数据库，需先导入属性后再预览选项 |

建议真实导入顺序：

1. 导入商品分类。
2. 导入商品属性。
3. 重新预览商品属性选项。
4. 预览通过后导入商品属性选项。

## 7. 待确认事项

如果预览通过，下一步需要用户确认是否把这些压测数据写入当前激活数据库。
