# 上游系统管理布局分析

日期：2026-06-04

## 结论

截图和旧项目的核心不是视觉样式，而是信息分布方式：

1. 左侧是主仓接入列表，用于切换当前主仓。
2. 右上是当前主仓的基本信息和主操作。
3. 右下是当前主仓的配置内容，按 Tab 切换仓库配对、物流渠道、SKU 配对、请求日志等。

当前若依首版已经有“选中主仓后右侧内容变化”的数据关系，但界面还是“上方接入表格 + 下方详情 Tab”。下一步应改成“左侧主仓列表 + 右侧详情”的工作台布局。

## 旧项目布局观察

旧项目文件：

- `E:\Urili\apps\admin-web\src\features\upstream-systems\UpstreamSystemManagementPage.tsx`
- `E:\Urili\apps\admin-web\src\styles.css`

旧项目三块分布：

| 区域 | 作用 | 可参考点 | 不照搬点 |
| --- | --- | --- | --- |
| 左侧主仓列表 | 搜索、按系统分组、选择主仓、新增主仓、调整顺序 | 左侧列表作为主入口，选中后驱动右侧变化 | 不照搬它的按钮样式、卡片样式、颜色、间距 |

| 右下配置 Tabs | 仓库、物流渠道、SKU、箱规、增值服务等配置随主仓变化 | Tabs 是当前主仓下的二级配置 | 第一版只落地已有功能，不提前做箱规/增值服务 |

## 当前若依首版现状

当前文件：

- `react-ui/src/pages/UpstreamSystem/index.tsx`
- `react-ui/src/pages/UpstreamSystem/components/SyncTabs.tsx`

当前结构：

1. 顶部 `ProTable` 展示所有主仓接入。
2. 点击表格行后设置 `selectedConnection`。
3. 下方 `SyncTabs` 根据 `selectedConnection.connectionCode` 拉取仓库、物流渠道、SKU、请求日志。

也就是说，数据流已经具备：

- 当前主仓状态：`selectedConnection`
- 右侧内容查询参数：`selectedConnection.connectionCode`
- 切换主仓后内容变化：已具备

缺口是布局表达：

- 主仓列表不应该作为顶部大表格占据主视觉。
- 右侧需要有明确的当前主仓信息区。
- 操作按钮应绑定当前主仓，而不是散落在主仓表格每一行。

## 建议的若依化布局

保持我们自己的 Ant Design Pro / 若依风格，不抄截图视觉，只调整空间结构。

```text
PageContainer
└── 上游系统管理工作台
    ├── 左侧：主仓列表栏
    │   ├── 标题：主仓接入
    │   ├── 搜索框：搜索主仓名称/接入编号
    │   ├── 新增按钮
    │   └── 列表：按上游系统类型分组
    │       └── 主仓项：名称、编号、状态、最近同步
    └── 右侧：主仓详情区
        ├── 顶部：基本信息和操作
        │   ├── 当前主仓名称 / 接入编号 / 状态
        │   ├── Key 脱敏值 / 结算类型 / 最近授权 / 最近同步
        │   └── 操作：编辑、授权、校验、同步、启停
        └── 底部：配置 Tabs
            ├── 领星仓库同步清单
            ├── 领星物流渠道同步清单
            ├── 领星SKU同步清单
            └── 请求日志
```

## 交互规则

- 页面加载后默认选中第一个主仓；如果没有主仓，右侧展示空状态和“新增主仓接入”按钮。
- 左侧点击不同主仓后：
  - 右上基本信息立即变为所选主仓。
  - 右下 Tabs 使用所选主仓 `connectionCode` 重新查询。
  - 当前选中的 Tab 可以保留；如果查询失败，在当前 Tab 内展示错误状态。
- 左侧搜索只过滤主仓列表，不影响右侧已经选中的主仓；如果搜索后当前主仓不可见，右侧仍保持当前主仓，避免误切换。
- 新增/编辑/授权/同步成功后刷新左侧列表，并保持当前主仓选中。
- 启停后刷新左侧状态和右侧基本信息。

## 字段展示建议

左侧主仓项保持轻量：

- 主仓名称
- 接入编号
- 状态
- 最近同步时间

右上基本信息展示完整但不拥挤：

- 主仓名称
- 接入编号
- 状态
- 上游系统类型
- 结算类型
- Key 脱敏值
- 最近授权时间
- 最近同步时间
- 同步状态摘要：仓库数、渠道数、SKU 数可后续补充

右下 Tabs 沿用当前已实现能力：

- 仓库同步清单：保留配对/解除配对。
- 物流渠道同步清单：保留同一领星渠道配多个系统渠道。
- SKU 同步清单：保留分页、搜索、配对/解除配对。
- 请求日志：保留只追加日志查看。

## 不建议照搬的点

- 不照搬截图里的浅灰大背景、旧项目按钮排布、旧项目左侧文字树样式。
- 不把“编辑主仓、同步、重新授权、删除”全部放成顶部右侧一排密集按钮；若依里应按当前权限和危险程度分组，删除/停用保持二次确认。
- 不把“箱配对、增值服务配对”空 Tab 先放出来；当前第一版没有正式表和业务规则，提前展示会误导用户。
- 不把旧项目的“映射”文案搬过来；当前已确认文案使用“配对”和“同步清单”。

  - 把当前接入 `ProTable` 改为左侧主仓列表数据源。
  - 保留新增、编辑、授权、同步、启停逻辑。
- 新增或调整组件：
  - `components/ConnectionSidebar.tsx`
  - `components/ConnectionSummary.tsx`
  - 保留 `components/SyncTabs.tsx`
- `react-ui/src/pages/UpstreamSystem/types.ts`
  - 补充左侧列表项 props 类型。

后端暂不需要调整：

- 当前 `/list`、`/{connectionCode}/warehouses`、`/{connectionCode}/logistics-channels`、`/{connectionCode}/skus/list`、`/{connectionCode}/request-logs/list` 已满足左选右变。
- 如果后续主仓很多，再考虑新增轻量左侧列表接口，避免左侧拉取完整分页表格字段。

## 验证重点

- 左侧切换主仓后，右侧基本信息和所有 Tab 查询都跟着切换。
- 新增主仓后自动选中新主仓。
- 同步完成后左侧最近同步时间、右侧同步清单同步刷新。
- 无权限用户不显示新增/编辑/授权/同步/配对按钮，后端仍拒绝无权限访问。
- 移动端或窄屏时左侧列表应变成上方选择区或可折叠区域，避免表格横向挤压。
