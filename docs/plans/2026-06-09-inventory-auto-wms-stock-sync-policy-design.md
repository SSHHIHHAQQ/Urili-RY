# 库存总览自动同步 WMS 库存策略设计方案

日期：2026-06-09

状态：待确认，未开始实现。

## 背景

当前库存总览已经支持手动调整 `SKU + 仓库` 明细行的平台总库存和平台在途库存。

新增需求是：部分卖家不想逐个商品维护平台总库存，而是希望 WMS 上有多少可用库存，就自动放多少到平台销售。例如 WMS 可用库存从 1000 变为 980，平台总库存也自动变为 980；后续 WMS 到货后可用库存变为 1200，平台总库存也自动变为 1200。

该能力面向两类卖家：

- 卖家不再自己销售，希望 WMS 里的可用库存全部放到本平台销售。
- 卖家多个销售渠道共享同一个 WMS 库存池，不想单独给本平台拆分库存上限。

页面文案采用卖家更容易理解的名称：`自动同步 WMS 库存`。

底层设计仍使用来源系统中立模型，不把表结构和服务命名绑定到某一个 WMS 或领星系统，避免后续接入 ERP、自研仓储系统或其他来源库存系统时返工。

## 目标

1. 库存总览显示卖家信息，并支持按卖家筛选。
2. 库存总览支持配置库存同步方式：
   - `手动设置平台库存`
   - `自动同步 WMS 库存`
3. 支持按卖家、仓库、SPU、SKU、SKU + 仓库明细行设置库存同步方式。
4. 来源库存同步后，自动同步模式下的平台总库存跟随 WMS 可用库存变化。
5. 自动同步引发的平台总库存变化必须写库存流水。
6. 保留平台自己的锁定库存、防超卖和来源扣减校准逻辑。

## 不做范围

第一版不做：

- 不把 WMS 在途库存直接加入平台总库存。
- 不对三方仓做自动同步 WMS 库存，除非后续三方仓也接入来源库存快照。
- 不绕过平台订单锁定库存。
- 不让前端自行计算库存同步结果。
- 不做买家侧展示文案调整。
- 不做跨卖家的批量策略设置。

## 库存口径

### 手动设置平台库存

保持当前逻辑。

用户手动维护平台总库存：

```text
平台可售库存 = max(0, 平台总库存 - 平台锁定库存)
```

官方仓仍受来源有效可用库存约束：

```text
来源有效可用库存 = max(0, 来源可用库存 - 待来源扣减)
```

如果平台总库存大于来源有效可用库存，官方仓可售计算仍不能突破来源有效可用库存。

### 自动同步 WMS 库存

自动同步 WMS 库存只同步 WMS 可用库存，不把 WMS 在途库存直接计入平台总库存。

计算口径：

```text
来源有效可用库存 = max(0, 来源可用库存 - 待来源扣减)

平台总库存 = 来源有效可用库存

平台可售库存 = max(0, 平台总库存 - 平台锁定库存)
```

如果来源有效可用库存小于平台锁定库存：

- 不阻塞自动同步。
- 平台总库存仍更新为来源有效可用库存。
- 平台可售库存为 0。
- 库存状态标记为库存风险，例如 `来源库存不足`。
- 写库存流水，便于追溯。

## 适用范围

`自动同步 WMS 库存` 只适用于有来源库存快照的官方仓库存行。

不支持自动同步的行：

- `NO_WAREHOUSE` 仓库未配置行。
- 来源 SKU 未绑定行。
- 无来源仓库库存快照的官方仓行。
- 未接入来源库存的三方仓行。

这些行在预览时必须显示跳过原因，不能静默忽略。

## 策略层级

需要支持五种设置范围：

| 设置范围 | 业务含义 | 典型场景 |
| --- | --- | --- |
| 卖家默认 | 该卖家所有商品默认使用某种库存同步方式 | 卖家希望全部商品都自动同步 WMS 库存 |
| 仓库设置 | 该卖家某个仓库下所有 SKU 使用某种库存同步方式 | CA012 全部自动同步，NY001 全部手动 |
| SPU 设置 | 该卖家某个 SPU 下所有 SKU 和仓库使用某种库存同步方式 | 某个产品整体自动同步 |
| SKU 设置 | 该卖家某个 SKU 下所有仓库使用某种库存同步方式 | 某个规格单独自动同步 |
| 明细行设置 | 某个 SKU + 仓库库存行单独设置 | 某 SKU 在某仓库例外处理 |

## 策略优先级

策略可能同时命中同一库存行，必须有确定优先级。

建议优先级：

```text
SKU + 仓库明细行设置
> 仓库设置
> SKU 设置
> SPU 设置
> 卖家默认
> 系统默认
```

系统默认：

```text
手动设置平台库存
```

仓库设置高于 SKU / SPU 设置的原因：

- 仓库是履约和来源库存边界。
- 如果卖家明确设置某仓库不自动同步，不能被某个 SKU 或 SPU 的通用设置绕过。
- 如果确实需要某 SKU 在该仓库例外自动同步，应设置更精确的 `SKU + 仓库明细行设置`。

## 卖家信息设计

库存总览三种视图都需要补充卖家信息。

后端读模型需要保证聚合维度带上 `seller_id`，避免不同卖家的相同 SKU 或相同商品名被错误合并。

建议字段：

- `seller_id`
- `seller_name`
- `seller_code`

页面改动：

- SPU 视图增加 `卖家` 列。
- SKU 视图增加 `卖家` 列。
- 仓库视图增加 `卖家` 列。
- 筛选区增加 `卖家` 下拉，支持模糊搜索。

## 数据模型方案

### 新增策略表

建议新增：

```text
inventory_stock_sync_policy
```

用途：

- 保存卖家、仓库、SPU、SKU、明细行维度的库存同步方式。
- 影响当前已有库存行。
- 影响未来新生成的库存行。

建议字段：

| 字段 | 类型建议 | 含义 |
| --- | --- | --- |
| `policy_id` | bigint | 主键 |
| `policy_key` | varchar(200) | 策略自然键，用于唯一约束 |
| `seller_id` | bigint | 卖家 ID |
| `scope_type` | varchar(32) | `SELLER` / `WAREHOUSE` / `SPU` / `SKU` / `SKU_WAREHOUSE` |
| `warehouse_key` | varchar(200) | 仓库维度自然键 |
| `warehouse_name` | varchar(100) | 仓库展示名 |
| `spu_id` | bigint | SPU ID |
| `sku_id` | bigint | SKU ID |
| `stock_key` | varchar(200) | SKU + 仓库明细行自然键 |
| `sync_mode` | varchar(32) | `MANUAL` / `AUTO_SOURCE_AVAILABLE` |
| `enabled` | char(1) | 是否启用 |
| `remark` | varchar(500) | 备注 |
| `version` | bigint | 乐观锁版本 |
| `create_by` / `create_time` | 若依通用字段 | 创建信息 |
| `update_by` / `update_time` | 若依通用字段 | 更新信息 |

`sync_mode` 不建议在数据库里叫 `AUTO_WMS`。

原因：

- 页面可以展示 `自动同步 WMS 库存`。
- 数据库和服务层保存 `AUTO_SOURCE_AVAILABLE`，表达的是“自动同步来源有效可用库存”。
- 后续来源系统不是 WMS 时，不需要改表和历史数据。

`policy_key` 生成规则建议：

```text
SELLER:{sellerId}
WAREHOUSE:{sellerId}:{warehouseKey}
SPU:{sellerId}:{spuId}
SKU:{sellerId}:{skuId}
SKU_WAREHOUSE:{sellerId}:{skuId}:{warehouseKey}
```

唯一约束：

```text
uk_inventory_stock_sync_policy_key(policy_key)
```

### 扩展库存明细读模型

建议扩展：

```text
inventory_sku_warehouse_stock
```

新增冗余字段：

| 字段 | 含义 |
| --- | --- |
| `sync_mode` | 当前生效库存同步方式 |
| `sync_policy_id` | 当前命中的策略 ID |
| `sync_policy_scope` | 当前命中的策略范围 |
| `sync_policy_key` | 当前命中的策略自然键 |
| `sync_status` | 自动同步状态，例如 `NORMAL` / `UNSUPPORTED` / `SOURCE_INSUFFICIENT` |
| `last_auto_sync_time` | 最近一次自动同步平台总库存时间 |

冗余原因：

- 库存总览列表不能每次实时多层匹配策略。
- 读模型刷新时统一计算生效策略，列表直接展示。
- 方便筛选 `手动设置平台库存` / `自动同步 WMS 库存`。

### 库存流水

继续复用：

```text
inventory_stock_ledger
```

自动同步导致平台总库存变化时写流水：

- `operation_source`：`SYSTEM`
- `biz_type`：`AUTO_WMS_STOCK_SYNC`
- `operation_type`：库存调整或系统同步，按现有枚举收敛
- `reason`：例如 `自动同步 WMS 库存`
- 记录调整前后平台总库存、平台可售库存、平台锁定库存。

如果后续需要更强追溯，可增加：

- `sync_policy_id`
- `sync_policy_scope`
- `sync_policy_key`

## 后端接口方案

建议新增库存同步策略接口，归属库存总览管理端。

基础路径：

```text
/inventory/admin/overview/sync-policy
```

接口：

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `POST` | `/preview` | 预览策略变更影响 |
| `POST` | `/confirm` | 确认策略变更并应用 |
| `GET` | `/list` | 查询策略列表，后续可用于设置弹窗回显 |
| `GET` | `/effective` | 查询某库存行当前生效策略，后续可选 |

权限：

```text
inventory:overview:syncPolicy
```

审计：

- `@Log(title = "库存同步方式设置", businessType = BusinessType.UPDATE)`
- 策略变更必须记录操作人、时间和备注。

### Preview 请求

请求字段：

| 字段 | 含义 |
| --- | --- |
| `sellerId` | 卖家 ID，必填 |
| `scopeType` | 设置范围 |
| `warehouseKey` | 仓库设置或明细行设置时需要 |
| `spuId` | SPU 设置时需要 |
| `skuId` | SKU 或明细行设置时需要 |
| `stockKey` | 明细行设置时可选，优先用 `skuId + warehouseKey` |
| `syncMode` | `MANUAL` / `AUTO_SOURCE_AVAILABLE` |
| `remark` | 备注，选填 |

Preview 返回：

| 字段 | 含义 |
| --- | --- |
| `affectedRowCount` | 命中的库存明细行数 |
| `eligibleRowCount` | 可应用行数 |
| `skippedRowCount` | 跳过行数 |
| `changedRowCount` | 平台总库存会发生变化的行数 |
| `beforePlatformTotalQty` | 调整前平台总库存合计 |
| `afterPlatformTotalQty` | 调整后平台总库存合计 |
| `rows` | 明细预览和跳过原因 |

### Confirm 规则

确认后：

1. Upsert 策略表。
2. 重新计算当前受影响库存行的生效策略。
3. 如果策略为 `AUTO_SOURCE_AVAILABLE`，立即按当前来源有效可用库存同步平台总库存。
4. 如果策略为 `MANUAL`，只切换后续模式，不强行回滚平台总库存；当前平台总库存保持最后一次值。
5. 写策略变更日志。
6. 对平台总库存发生变化的行写库存流水。
7. 刷新 SKU / SPU 聚合读模型。

Confirm 不允许部分成功。

如果存在不允许应用的行：

- Preview 展示跳过原因。
- Confirm 可以只对 eligible 行执行，但必须在预览里明确 “将跳过 N 行”。
- 如果用户选择的精确范围是明细行，且该行不 eligible，则 Confirm 直接失败。

第一版建议：卖家、仓库、SPU、SKU 范围允许跳过不支持行；明细行设置不允许跳过。

## 来源库存刷新链路

来源仓库库存同步落库后，库存总览刷新应增加自动同步步骤。

建议顺序：

```text
1. 刷新 inventory_sku_warehouse_stock 明细行
2. 计算每条库存行生效的库存同步策略
3. 对 AUTO_SOURCE_AVAILABLE 行同步平台总库存
4. 写自动同步库存流水
5. 刷新 SKU / SPU 聚合读模型
```

注意：

- 手动模式不能被来源库存刷新覆盖平台总库存。
- 自动模式只有平台总库存发生变化时才写流水。
- 来源有效可用库存小于平台锁定库存时不阻塞同步，但标记风险。
- 策略计算必须在读模型刷新后执行，确保新生成的库存行也能继承卖家或仓库默认策略。

### 顶部按钮

按钮名称：

```text
自动同步 WMS 库存设置
```

按钮位置：

- 库存总览 ProTable 原生 toolbar。
- 不放到筛选区。
- 不在表格上方另起独立设置区。
- 不做页面级标题或说明区。
- 与现有 `SPU视图 / SKU视图 / 仓库视图` 切换控件放在同一个 toolbar。

视觉顺序建议：

```text
[SPU视图 | SKU视图 | 仓库视图]  [自动同步 WMS 库存设置]
```

原因：

- 视图切换是当前表格的查看方式，继续放在 toolbar。
- `自动同步 WMS 库存设置` 是跨范围配置动作，作为 toolbar 右侧主操作按钮。
- 不进入筛选区，避免用户误以为它只是筛选条件。
- 不额外做独立按钮区，避免破坏库存总览当前紧凑列表页结构。

权限：

- 需要 `inventory:overview:syncPolicy`。
- 无权限时不展示按钮。

三视图一致：

- SPU 视图、SKU 视图、仓库视图都显示同一个 toolbar 按钮。
- 仓库视图通过 `toolbar` prop 复用同一组 toolbar 控件，避免三视图按钮位置不一致。

弹窗内容：

1. 设置范围：
   - 按卖家
   - 按仓库
   - 按 SPU
   - 按 SKU
   - 按明细行
2. 选择卖家。
3. 根据设置范围选择仓库、SPU、SKU 或明细行。
4. 选择库存同步方式：
   - 手动设置平台库存
   - 自动同步 WMS 库存
5. 备注，选填。
6. 点击 `预览影响`。
7. 展示影响范围和跳过原因。
8. 点击 `确认应用`。

```text

```

#### 第二段：预览影响

点击 `预览影响` 后，在同一个 Modal 内展示预览结果，不跳新页面，不另开抽屉。

预览摘要使用紧凑信息条或 `Descriptions`：

```text
命中库存行：120
可应用：116
跳过：4
平台总库存变化：980 -> 1260
```

如果存在跳过行，展示 Ant Design `Alert`：

```text
有 4 条库存不支持自动同步 WMS 库存，将不会应用。本次只会应用可处理的库存行。
```

预览表格使用 Ant Design `Table`，固定高度滚动，不撑高弹窗。

预览表字段：

| 字段 | 含义 |
| --- | --- |
| SKU | 系统 SKU 和 SKU 名称 |
| 仓库 | 仓库名和仓库类型 |
| 当前同步方式 | 当前生效方式 |
| 规则来源 | 当前生效规则来源 |
| 新同步方式 | 本次要设置的方式 |
| 当前平台总库存 | 当前值 |
| 同步后平台总库存 | 自动同步时的目标值；切回手动时显示保持当前值 |
| 处理结果 | 可应用 / 跳过 |
| 原因 | 跳过原因或风险提示 |

预览表里的状态用 Tag：

```text
可应用
跳过
库存风险
```

第二段底部按钮：

```text
重新预览
确认应用
```

规则：

- 表单项变更后，旧预览失效，`确认应用` 禁用，必须重新预览。
- `确认应用` 按钮文案可以带数量，例如 `确认应用 116 条`。
- 如果可应用行数为 0，`确认应用` 禁用。
- 如果是明细行精确设置且该行不可应用，直接显示错误并禁用确认。

#### 弹窗关闭与刷新

确认成功后：

- 关闭弹窗。
- 刷新当前视图表格。
- 如果当前展开行仍存在，第一版可以不保持展开状态，优先保证数据刷新正确。

取消或关闭时：

- 清空预览结果。
- 不保留未提交表单状态。

#### 与现有库存调整弹窗的区别

现有 `调整库存` 弹窗是直接编辑库存数值。

`自动同步 WMS 库存设置` 弹窗是配置策略，不直接让用户输入库存数量。

因此：

- 不展示 `调整后平台总库存` 输入框。
- 不允许前端自行计算并保存库存。
- 必须先 Preview，再 Confirm。
- Confirm 后由后端决定哪些库存行同步平台总库存，哪些库存行跳过。

## 与现有库存调整的关系

现有 `调整库存` 能力继续保留。

但自动同步模式下：

- 平台总库存不建议允许直接手动编辑。
- 如果用户要手动改平台总库存，应先切换为 `手动设置平台库存`。
- 平台在途库存是否允许继续手动调整，第一版保持现状，后续再决定是否也做自动同步。

页面提示建议：

```text
当前库存行已开启自动同步 WMS 库存。若需手动调整平台总库存，请先切换为手动设置平台库存。
```

## 风险与边界

### WMS 文案与来源系统中立

页面使用 `WMS` 是为了让卖家理解。

后端表、枚举和服务不直接使用 `WMS`，避免未来接入非 WMS 来源系统时产生命名债。

### 多仓库策略冲突

仓库设置优先级高于 SKU / SPU 设置。

如果卖家需要某个 SKU 在某仓库例外，应设置 `SKU + 仓库明细行`。

### 未来新库存行

卖家默认和仓库设置必须影响未来新生成的库存行。

因此不能只给当前 `inventory_sku_warehouse_stock` 行批量写模式字段，必须有策略表。

### 跳过行必须可见

批量设置时，不能静默跳过不支持自动同步的行。

必须在 Preview 里展示：

- 跳过 SKU。
- 跳过仓库。
- 跳过原因。
